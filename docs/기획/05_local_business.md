- 확장 표기(문맥 필요 시): "수원·서울 등 수도권과 충청권에서도 방문하십니다."
- ⚠️ 충주·제천·원주·음성 등은 잘못된 정보 — 어떤 페이지에도 사용 금지.

## 3. 구조화 데이터 (JSON-LD)

전 페이지 공통으로 `AutoRepair`(LocalBusiness 하위) 스키마를 삽입한다:

```json
{
  "@context": "https://schema.org",
  "@type": "AutoRepair",
  "name": "모터리페어",
  "alternateName": "장용석의 내차를 부탁해 동탄신도시점",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "효행로1156번길 9",
    "addressLocality": "화성시",
    "addressRegion": "경기도",
    "addressCountry": "KR"
  },
  "telephone": "+82-10-3203-7607",
  "openingHoursSpecification": [
    { "@type": "OpeningHoursSpecification", "dayOfWeek": ["Monday","Tuesday","Wednesday","Thursday","Friday"], "opens": "08:30", "closes": "18:30" },
    { "@type": "OpeningHoursSpecification", "dayOfWeek": "Saturday", "opens": "08:30", "closes": "15:00" }
  ],
  "areaServed": ["동탄", "화성", "오산", "평택"]
}
```

- 문의 페이지에는 FAQPage 스키마 추가 (page_문의예약 참조).
- 좌표(geo)는 빌드 시 실제 좌표로 채운다.

## 4. 로컬 SEO 체크리스트

**사이트 쪽:**
- [ ] NAP·영업시간이 §1 기준값과 전 페이지 일치 (상수 파일 단일 관리)
- [ ] 푸터에 서비스 지역 텍스트 노출 ("동탄·화성·오산·평택 수입차 정비 — 브랜드 6개")
- [ ] 오시는길 페이지에 지역별 접근 텍스트 (자기완결 문장)
- [ ] JSON-LD AutoRepair 전 페이지 삽입
- [ ] 전화번호 tel: 링크 (헤더·푸터·문의·플로팅)

**네이버 스마트플레이스 쪽:**
- [ ] 상호·주소·전화·영업시간이 §1과 글자 단위 일치
