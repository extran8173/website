# 04_신뢰섹션.md

> 저장 위치: 리포지토리 `docs/기획/04_신뢰섹션.md`
> 역할: 사이트 전체에 **분산 배치**되는 신뢰 요소의 원본 정의. 각 페이지 MD는 이 문서의 요소를 가져다 쓴다. 문구를 수정할 때는 여기를 고치고 각 페이지에 반영한다(단일 소스).

---

## 1. 보증 / A/S

- **표준 문구:** `교환한 부품은 차량 제작사 기준과 동일하게 1년 / 2만km 동안 A/S를 적용합니다.`
- **보증 중 차량 안내 문구:** `보증 기간 중인 차량은 입고 시 미리 알려주세요. 보증 수리 대상 항목은 공식 서비스센터 이용이 유리할 수 있어 사전에 안내해 드립니다.`
- 배치: trust-badge-row(메인·문의) · 서비스 §2 caption · 진단장비 §5 · FAQ 1·7 · 푸터 법정 행

## 2. 정비 과정 (입고 → 점검 → 정비 → 출고)

- **표준 3단계 문구** (서비스 §2 원본):
  1. 진단 — 전용 진단기와 계측 장비로 원인을 먼저 확인합니다.
  2. 견적 안내 — 교환이 필요한 부위를 사진으로 보여드리고, 작업 전에 견적을 안내합니다. 승인 후에 작업합니다.
  3. 정비·기록 — 정비 과정을 사진으로 기록하고, 출고 시 함께 설명해 드립니다.
- 이 과정 자체가 B2(바가지)·Q10(불필요 수리)의 구조적 해답 — "승인 없이는 작업하지 않는다"가 핵심.
- 배치: 서비스 §2(원본) · 문의 첫 방문 3단계(고객 시점 변형) · 각 정비 사례 글(사진 시퀀스로 매주 실증)

## 3. 사회적 증거

- **구글 평점:** ★4.7 — 텍스트로 표기. 리뷰 수(7건)는 표기하지 않는다(거짓 없이, 약점 강조 없이).
- **후기 인용 3개** (page_메인 §7 — 빌드 전 캡처 원문 대조 필수):
  - "간단한 작업이라도 하나하나 꼼꼼히 체크해 주셔서 항상 믿고 차를 맡길 수 있어요."
  - "직접 진단에 신경을 많이 쓰는 곳이에요. 자세히 설명해 주셔서 좋았습니다." — BMW 320d 차주
  - "정비소 많이 다녀봤는데 진짜 무언가 다른 곳입니다."
- **주간 정비 사례 = 1차 증거.** 후기보다 강하다(사진·원인·결과 포함). 사회적 증거 전략의 중심은 후기 수집이 아니라 사례 누적이다.
- 배치: 메인 §7(후기) · recent-cases(메인·서비스·진단장비) · 정비 사례 페이지 전체
- 운영 권장: 네이버 개별 후기 선명 캡처 재확보 시 review-strip에 출처 다변화(구글+네이버).

## 4. 인증 / 자격 / 경력

- **표준 문구(경력):** `대표 장용석은 수입차 서비스센터와 엔진 전문회사에서 20년 넘게 정비해 왔습니다.`
- **표준 문구(인증):** `모터리페어는 ISO 9001(품질경영) · ISO 14001(환경경영) · ISO 45001(안전보건경영) 인증을 취득했습니다.`
- 표기 규칙: 인증은 반드시 3종 모두, "취득"으로 표기. 괄호 안 쉬운 설명 유지. 이미지가 아닌 텍스트로(GEO).
- 배치: trust-badge-row · 진단장비 §4~5(집중) · 소개 문단(메인 §3) · 푸터
- ISO 인증서 실물 사진 확보 시: 진단장비 §5에 추가 (IMG-012).

## 5. 대표자 스토리

- 현재 확보된 사실: 20년 경력(서비스센터·엔진 전문회사), 직접 진단·판단, "고급스럽고 담담하게 정비"라는 지향.
- **현 단계 표현 원칙:** 별도 스토리 페이지를 만들지 않는다. 스토리는 진단장비 §4("장비는 도구입니다. 판단은 경력이 합니다") 한 섹션과, 매주 사례 글의 담담한 서술 문체 자체로 전달한다 — 말하는 스토리가 아니라 보여주는 스토리.
- 대표 프로필 사진 확보 시: 진단장비 §4에 추가. 그 후에도 별도 "대표 인사말" 페이지는 만들지 않는다(브랜드 톤과 충돌 — 과장 없는 담담함 유지).

## 6. 분산 배치 지도 (최종 점검표)

| 위치 | 배치 신뢰 요소 |
|---|---|
| 메인 히어로 직하단 | trust-badge-row (경력·ISO·A/S) |
| 메인 §3 소개 | GEO 엔티티 선언 + 경력 문장 |
| 메인 §6~7 | 최근 사례 + 후기 |
| 서비스 §2 | 정비 과정 3단계 + A/S caption |
| 서비스 각 항목 | 항목별 근거 한 줄 |
| 진단장비 전체 | 물증 집중(장비 사진·경력·ISO) |
| 정비 사례 | 살아있는 증거 (주간 갱신) |
| 문의 §1 | trust-badge-row (마지막 보강) |
| 푸터 | ISO·A/S·지역 텍스트 (전 페이지) |

- 원칙 재확인: 신뢰 요소는 한 곳에 몰지 않는다. 같은 요소가 세 페이지 이상에서 같은 형태로 반복되면 조정한다.
