# 02_가치_궁금증_분석

> 저장 위치: 리포지토리 `docs/기획/02_가치_궁금증_분석.md`
> 역할: 모든 페이지 카피는 이 문서의 가치·궁금증·장벽 매핑에서 출발한다. 페이지별 MD 작성 시 이 문서의 항목 번호(Q1, B1 등)를 참조해 누락을 막는다. (최종 점검은 07_소구포인트_누락확인표에서 수행)

## 1. 핵심·차별화 가치 (우선순위)

**핵심 가치 (한 문장):**
> 동탄에서 20년 경력 정비사가 브랜드 전용 진단기로 원인을 정확히 짚고, 작업 과정을 사진으로 투명하게 공유하는 수입차 전문 정비소.

**차별화 가치 우선순위 + 사이트 반영 위치:**

| 순위 | 가치 | 근거·입증 자산 | 주 반영 위치 |
|---|---|---|---|
| 1 | **정확한 진단력** | ISTA+(BMW)·Xentry(벤츠)·ODIS(아우디)·PIWIS(포르쉐)·Picoscope 풀라인 + 20년 경력. 진단 노트북 세트·Picoscope·PIWIS 화면 사진 보유 | 진단 장비 페이지 전면, 메인 중단 |
| 2 | **투명성 (가장 강한 전환 무기)** | 입고→점검→정비→출고 일관 포맷 + 사진 기록. 모니터 설명 컷 보유. 주간 사례 글이 누적 증거 | 정비 사례 전체, 메인 |
| 3 | **친절·꼼꼼한 설명** | 후기 반복 키워드("하나하나 꼼꼼히", "자세히 설명"). 구글 리뷰 캡처 보유 | 후기 섹션, 소개 |
| 4 | **근거리·빠른 응대** | 동탄·화성·오산·평택 접근성 | 오시는 길, 메인 히어로(지역 명시) |
| 5 | **객관적 신뢰 근거** | ISO 9001·14001·45001 취득, A/S 1년/2만km — 영세 사설 정비소가 드물게 갖춘 자산 | 신뢰 배지(전 페이지 분산), 진단 장비 페이지 |

**서비스별 가치:**

| 서비스 | 고객 이점 | 대응 궁금증 |
|---|---|---|
| 진단(경고등·증상) | 원인을 정확히 짚어줌 | Q10 (불필요한 수리 권하진 않을까) |
| 오일류 | 딜러 대비 합리적 비용 + 수입차용 규격 | Q6 (싸도 품질 괜찮나) |
| 엔진·미션(중정비) | 브랜드 경험 기반 수리 | Q2 (사설에서 되나) |
| 브레이크·서스펜션 | 소음·승차감 문제 해결 | Q11 (상태를 정직하게 알려줄까) |
| 냉·난방 | 에어컨·냉각수 등 계절 문제 | Q3 (한 번에 잡아줄까) |

**경쟁 포지션 유의점:** 리뷰 수가 적어(★4.7 / 7건) 사회적 증거가 얇다. → **주간 정비 사례 글이 이 약점을 메우는 핵심 수단**이다. 사례 1편 = 후기 1건 이상의 신뢰 가치(사진·원인·결과가 있으므로). 모든 페이지에서 사례로 향하는 보조 동선을 유지한다.

## 2. 고객 궁금증 전체 목록 + 페이지·섹션 매칭

| # | 궁금증 (고객 언어) | 유형 | 해소 페이지 · 섹션 | 해소 방식 |
|---|---|---|---|---|
| Q1 | "내 차 브랜드도 되나요?" (BMW·벤츠·아우디·폭스바겐·미니·포르쉐) | 서비스 | 메인 히어로 직하단 · 진단 장비 | 브랜드 텍스트 명시 + 브랜드별 전용 진단기 매칭 표기 |
| Q2 | "엔진·미션 같은 큰 수리도 사설에서 되나요?" | 서비스 | 서비스(중정비 섹션) · 정비 사례 | 20년 경력(서비스센터·엔진 전문회사) + 실제 중정비 사례 글 링크 |
| Q3 | "경고등·냉각수·에어컨·소음·도어 이런 증상도 봐주나요?" | 서비스 | 메인(증상 진입 블록) · 서비스(증상별 앵커) | 증상 언어 그대로 나열("경고등이 떴어요" 등) → 해당 서비스로 연결 |
| Q4 | "공식센터보다 얼마나 싼가요? 품질 차이는요?" | 가격 | 서비스 · FAQ | 가격표 대신 "작업 전 견적 안내" 원칙 + 제작사 부품 기준 동일 A/S로 품질 답변 |
| Q5 | "견적은 미리 알려주나요?" | 가격 | 서비스 · FAQ · 문의 | "작업 전 견적 안내, 승인 후 작업" 프로세스 명시 |
| Q6 | "싸면 부품 품질이 떨어지는 건 아닌가요?" | 가격 | 서비스(오일류) · FAQ | 수입차용 규격 부품 사용 명시 + 사례 글의 부품 실명(arnott 등) |
| Q7 | "장비는 제대로 갖췄나요?" | 신뢰 | 진단 장비 페이지 | 장비 실명 + 실물 사진(진단 노트북·Picoscope·PIWIS) |
| Q8 | "정말 잘하나요? 후기는요?" | 신뢰 | 메인(후기) · 정비 사례 | 구글 리뷰 인용(★4.7) + 사례 글 누적으로 보강 |
| Q9 | "작업 과정을 사진으로 보여주나요?" | 신뢰 | 정비 사례 전체 · 메인 | 사례 글 자체가 답 — "모든 정비를 사진으로 기록합니다" 선언 + 모니터 설명 컷 |
| Q10 | "불필요한 수리를 권하진 않을까요?" | 신뢰 | 정비 사례 · FAQ | 사진 근거 견적("패드 잔량 사진 첨부") + 진단→승인→작업 순서 명시 |
| Q11 | "상태를 정직하게 알려줄까요?" | 신뢰 | 정비 사례(원인 섹션) · 후기 | 사례 포맷의 '원인' 항목이 매번 입증 + "꼼꼼히 체크" 후기 인용 |
| Q12 | "어디에 있나요? 동탄·오산·평택에서 가까운가요?" | 접근성 | 오시는 길 · 푸터 | 주소·지도 + 지역별 소요 시간 텍스트 |
| Q13 | "예약해야 하나요? 당일도 되나요?" | 접근성 | FAQ · 문의 | 전화·네이버 예약 안내 + 당일 가능 여부 정책 명시 (※사실 확인 필요 — 아래 '확인 필요' 참조) |
| Q14 | "처음 가는데 뭘 준비해야 하나요?" | 과정 | 문의(첫 방문 안내) · FAQ | 준비물·절차 3단계 안내 |
| Q15 | "수리 후 문제가 생기면요? (A/S)" | 과정 | FAQ · 신뢰 배지 · 서비스 | "제작사 부품 기준 동일, 1년/2만km" 텍스트 명시 |
| Q16 | "사설에서 수리하면 제조사 보증이 깨지나요?" | 과정 | FAQ (GEO 핵심 질문) | 보증 중 차량 사전 안내 원칙 + 자연어 Q&A로 작성 |

**매칭 원칙:** 하나의 궁금증은 주 해소 위치 1곳 + 보조 위치 1곳까지만. 같은 답을 세 곳 이상 반복하면 산만해진다. FAQ는 Q4·Q5·Q13·Q14·Q15·Q16(과정·가격형)을 담당하고, 신뢰형(Q7~Q11)은 콘텐츠 자체(장비 페이지·사례 글)로 답한다.

## 3. 신뢰 장벽 + 해소 전략

| # | 장벽 | 고객 불안 (원문 언어) | 해소 전략 | 적용 위치 |
|---|---|---|---|---|
| B1 | 보증 우려 | "딜러 아닌 곳에 맡기면 보증 깨지나" | 보증 중 차량 사전 안내 + 제작사 부품 기준 동일 A/S(1년/2만km) 명시 | FAQ(Q16) · 신뢰 배지 |
| B2 | 바가지 우려 | "바가지 안 쓰려면" | 작업 전 견적 + 사진 근거 제시 프로세스("교환 필요 부위를 사진으로 먼저 보여드립니다") | 서비스 · 정비 사례 |
| B3 | 실력 검증 불가 | "정말 잘하는 곳인지 어떻게 아나" | 전용 진단기 풀라인 실물 사진 + 20년 경력 + ISO 3종 — 말이 아닌 물증 | 진단 장비 페이지 상단 |
| B4 | 사회적 증거 부족 | "후기가 적어서 불안한데" | 주간 정비 사례 누적(차종+증상+원인+수리+결과+사진) — 후기보다 강한 1차 증거 | 정비 사례 · 메인 최근 사례 |
| B5 | 정보 부족 | "내 증상·차종이 대상인지 모르겠다" | 증상별 진입 블록(고객 언어) + 브랜드 명시 | 메인 · 서비스 |
| B6 | 절차 막막함 | "처음인데 뭘 어떻게 해야 하나" | 첫 방문 안내(준비물·절차) + 전화·네이버 예약 상시 노출 | 문의 · FAQ · 플로팅 |

**전략 핵심:** B2·B3·B4는 서로 다른 장벽이지만 **하나의 무기 — "사진으로 원인을 보여주는 투명성" — 이 동시에 해소**한다. 그래서 정비 사례가 사이트의 신뢰 엔진이고, 모든 페이지가 사례로 향하는 보조 동선을 갖는다(01 사이트맵의 보조 전환 설계와 일치).

**분산 배치 원칙:** 신뢰 요소를 한 페이지에 몰지 않는다.
- 메인 히어로 직하단: trust-badge-row (경력 20년 · ISO 3종 · A/S 1년/2만km)
- 서비스 각 항목: 해당 서비스의 근거 한 줄
- 진단 장비 페이지: 물증 집중(장비 사진 + 경력 + 인증)
- 정비 사례: 매주 갱신되는 살아있는 증거
- 문의 페이지: 마지막 설득 — 가장 강한 것 1~2개(사진 투명성 + A/S)

## 4. 확인 필요 (추측 금지 항목 — 사장님 답변 대기)

기획서에 사실로 박기 전에 확인이 필요한 항목이다. 페이지별 MD 작성 전까지 답을 받는다.

1. **당일 정비 가능 여부** (Q13) — "예약 우선, 당일은 전화 확인" 등 실제 운영 방식은?
2. **보증 중 차량 처리 방식** (Q16/B1) — 보증 수리 대상이면 공식센터 안내를 해주는지, 실제 안내 멘트는?
3. **영업시간·휴무일** — 푸터·오시는 길·네이버 플레이스에 100% 일치시켜야 하는 값.
4. **대차·픽업 서비스 유무** — 있다면 접근성 가치에 추가할 강한 소구점.
