# 03_공통UI설계

> 저장 위치: 리포지토리 `docs/기획/03_공통UI설계.md`
> 역할: 전 페이지에 공통 적용되는 UI(헤더·푸터·플로팅 CTA·공통 스타일)의 단일 기준. 시각 규격은 `DESIGN-apple.md` + `DESIGN-motorrepair-rules.md`를 따르고, 이 문서는 콘텐츠·구조·동작을 확정한다.

---

## 1. 헤더

### 데스크톱 (834px 초과)

```
[헤더] — sub-nav-frosted 규격 (parchment 80% + backdrop-blur, 높이 52px, 상단 고정 sticky)
  ├─ 좌: 가로형 로고 "MOTOR REPAIR" (홈 링크, 높이 ~24px)
  ├─ 중: 메뉴 6개 — 홈 · 서비스 · 정비 사례 · 진단 장비 · 오시는 길 · 문의
  │      (button-utility 타이포 14px, 현재 페이지는 ink 100% / 나머지는 ink-muted-80)
  └─ 우: [cta-call 필 버튼 "전화 상담"] + 전화번호 텍스트 "010-3203-7607"
         (전화번호는 caption 타이포, 클릭 시 tel: — 데스크톱에서도 링크 유지)
```

- 헤더 내 유일한 컬러 강조는 cta-call(Action Blue)이다. 로고·메뉴는 무채색.
- 스크롤 시에도 상단 고정(sticky). blur가 콘텐츠 위에 떠 있는 느낌을 만든다.

### 모바일 (834px 이하)

```
[헤더] — 동일 규격, 높이 52px
  ├─ 좌: 가로형 로고 (축소, 높이 ~20px)
  └─ 우: [전화 아이콘 버튼(44×44, tel: 탭 투 콜)] [햄버거 버튼(44×44)]
```

**햄버거 펼침(전체 화면 오버레이, canvas 배경):**
```
  ├─ 메뉴 6개 세로 스택 (body-strong 17px, 항목 높이 56px — 탭 여유)
  ├─ 구분 hairline
  └─ 하단 고정: [cta-call "전화로 증상 상담하기"] (채움 필)
              [cta-naver "네이버로 예약하기"] (고스트 필)
              버튼 높이 48px · 간격 12px · 좌우 여백 16px
```

- 펼침 상태에서 햄버거는 X(닫기)로 전환. 배경 스크롤 잠금.

## 2. 푸터 (전 페이지 공통)

원본 footer 규격(parchment 배경 · dense-link 2.41 행간 · 상하 패딩 64px). 모바일 1단 세로 스택, 데스크톱 3단.

```
[푸터]
  ├─ 열 1 — 사업자 정보
  │   ├─ 원형 엠블럼(24px) + 상호: 모터리페어
  │   ├─ 장용석의 내차를 부탁해 동탄신도시점
  │   ├─ 사업자등록번호 289-02-02283
  │   ├─ 주소: 경기도 화성시 효행로1156번길 9 (병점동 203-35)
  │   └─ 영업시간: (확인 필요 — 02 문서 §4-3, 네이버 플레이스와 일치시킬 것)
  ├─ 열 2 — 메뉴
  │   └─ 사이트 메뉴 6개 (dense-link)
  ├─ 열 3 — 연락·예약
  │   ├─ 전화 010-3203-7607 (tel: 링크, body-strong으로 유일하게 강조)
  │   ├─ 네이버 예약 링크 (새 창)
  │   └─ 네이버 블로그 blog.naver.com/extran (새 창)
  └─ 최하단 법정·SEO 행 (fine-print 12px, ink-muted-48)
      ├─ ISO 9001 · 14001 · 45001 취득
      ├─ 부품 A/S: 차량 제작사 기준 동일 (1년 / 2만km)
      └─ "동탄 · 화성 · 오산 · 평택 수입차 정비 — BMW · 벤츠 · 아우디 · 폭스바겐 · 미니 · 포르쉐"
          (로컬 SEO + GEO용 자기완결 텍스트 — 이미지가 아닌 텍스트로)
```

## 3. 플로팅 CTA (모바일 전용, 1024px 미만)

`fab-contact` 컴포넌트 (디자인 적용규칙 §4 정의).

- 위치: 우측 하단 고정 (우 16px · 하 16px + 안전영역 inset).
- 기본 상태: 56×56px 원형 FAB, Action Blue 채움 + 전화 아이콘(white).
- 탭하면 위로 펼침: [전화 상담(tel:)] → [네이버 예약(새 창)] 2개 버튼, 각 48px 높이 + 텍스트 라벨 병기, 간격 12px. 다시 탭하거나 바깥 탭하면 접힘.
- **노출 규칙:** 스크롤 시작 후 표시(히어로에서는 히어로 CTA와 중복되므로 숨김). 푸터 도달 시 푸터 위로 밀려 올라가지 않게 z-index·마진 처리.
- **정비 사례 상세 페이지 예외:** `sticky-case-cta`(하단 고정 바)가 노출되는 동안 FAB은 숨긴다 — 하단 전환 요소는 항상 1개만.

## 4. 공통 스타일 가이드 (요약 — 상세는 디자인 문서)

| 항목 | 기준 |
|---|---|
| 액센트 | Action Blue #0066cc 단일. 모든 "누르세요" 신호는 이 색 하나 |
| 폰트 | SF Pro → Pretendard 폴백 스택 (적용규칙 §1). 본문 17px/1.47, 사례 장문만 1.6 |
| 배경 리듬 | canvas(white) ↔ parchment(#f5f5f7) ↔ dark tile 교차. 색 전환이 구분선 |
| 라운드 | 필 버튼 9999px · 카드/사진 18px · 유틸 8px. 풀블리드 타일은 0 |
| 그림자 | 없음 (다큐멘터리 사진 포함 전면 금지) |
| 터치 타깃 | 최소 44×44px, 버튼 간격 8px 이상 |
| 인터랙션 | 버튼 press 시 scale(0.95) — 시스템 유일 마이크로 인터랙션 |
| 이미지 | Astro 자동 최적화(WebP·srcset·lazy). 히어로만 eager 로드 |
| alt text | [지역+업종+설명] 형식. 장식용은 alt="" |

## 5. 공통 섹션 컴포넌트 (여러 페이지에서 재사용)

| 컴포넌트 | 내용 | 사용 페이지 |
|---|---|---|
| `trust-badge-row` | 경력 20년 · ISO 9001·14001·45001 · A/S 1년/2만km — pill 3개 가로 배열(모바일 세로) | 메인 히어로 직하단 · 문의 |
| `cta-section` | 페이지 하단 공통 전환 블록: 짧은 설득 문구 1줄 + cta-call + cta-naver | 전 페이지 하단 |
| `recent-cases` | 최근 정비 사례 카드 3개(모바일 1단/데스크톱 3단) — Astro 컬렉션 최신순 자동 | 메인 · 서비스 하단 |
| `case-card` | 사례 카드: 대표 사진 + 차종·증상 제목 + 결과 한 줄 + 날짜 (store-utility-card 규격) | 사례 목록 · recent-cases |
| `review-strip` | 구글 리뷰 인용 카드 2~3개 + "★4.7" 텍스트 | 메인 |

- `cta-section` 문구는 페이지 맥락에 맞게 페이지별 MD에서 지정한다 (예: 서비스 하단 "증상만 말씀하셔도 됩니다").

## 6. 접근성·품질 체크 (공통)

- 모든 인터랙티브 요소 키보드 포커스 가능, 포커스 링은 primary-focus 2px (원본 규격).
- 다크 타일 위 텍스트 명도 대비 4.5:1 이상 확인 (body-muted #cccccc는 대형 텍스트에만).
- 전화번호·주소·영업시간은 사이트 전체에서 단일 소스(상수 파일)로 관리 — 네이버 플레이스와 불일치 방지.
- 햄버거·FAB 등 아이콘 버튼에 aria-label 필수.
