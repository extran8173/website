## 페이지 목적

사이트의 신뢰 엔진. 주 1회 발행되는 사례가 누적되며 B2(바가지)·B3(실력)·B4(증거 부족)를 동시에 해소한다. 두 진입 흐름을 모두 감당한다 — ① 사이트 내부에서 신뢰 확인차 들어오는 비교형, ② 검색("차종+증상")으로 상세 글에 바로 떨어지는 외부 유입. 특히 ②는 그 페이지 하나에서 전환까지 완결돼야 한다.

## URL·슬러그 규칙 (확정)

- 목록: `/cases/` / 상세: `/cases/{slug}/`
- 슬러그: 영문 소문자·하이픈, `브랜드-모델-증상요약` 순 (예: `bmw-520d-coolant-leak`, `benz-e400-air-suspension`). 발행 스킬이 frontmatter에서 자동 생성하되 검토 단계에서 수정 가능.
- 한글 URL을 쓰지 않는 이유: 공유 시 퍼센트 인코딩으로 깨져 보이고, 파일 시스템·배포 환경 간 호환 문제가 있다. 한글 검색 노출은 타이틀·본문이 담당한다.

## SEO / 메타

**목록 페이지:**
- 타이틀: `수입차 정비 사례 | 동탄 모터리페어`
- 메타: `BMW·벤츠·아우디·포르쉐 정비 기록. 차종, 증상, 원인, 수리 과정과 결과를 사진과 함께 남깁니다.`

**상세 페이지:** frontmatter의 `seo_title`·`description`을 그대로 사용 (젬이 생성, 파이프라인 스키마 참조). 패턴: `차종+증상+수리 | 동탄 수입차 정비` — "차종+증상" 검색이 이 페이지들의 타깃 키워드다.

## 운영 연결 (요약 — 상세는 파이프라인 문서)

- 발행: 사진+젬 원고 드롭 → `publish-case` 스킬 → captions.md 검토 → push → Cloudflare 배포.
- 발행 즉시 자동 반영되는 곳: 이 목록 페이지, 메인·서비스의 recent-cases, 상세의 관련 사례. — 별도 수작업 없음.
- 태그·카테고리는 frontmatter 값을 그대로 쓴다. 목록 필터는 brand 필드 기준(카테고리 필터는 사례 30편 이상 시 검토).
